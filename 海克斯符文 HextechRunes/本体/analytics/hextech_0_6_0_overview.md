## 服务器修复记录

- 本次抓取前，公开 API 返回 502。服务器日志显示 Node 在启动时读取 513MB `run_results.jsonl` 为单个字符串，触发 `ERR_STRING_TOO_LONG`。
- 已将统计服务改为分块流式读取 JSONL，并给 summary 接口加入按数据文件 mtime 区分的缓存；页面和 summary API 不再因为数据量过大而整文件读入内存。
- 已同步最新中文标签到服务器统计 labels，避免 0.6.0 新符文在统计页显示为 ID。
- 为避免访问页面时触发超大 CSV 派生表重建，旧 derived CSV 不再因数据过期自动重建；缺失的 CSV 会返回 503，`summary.json` 则改为直接走流式实时汇总。
